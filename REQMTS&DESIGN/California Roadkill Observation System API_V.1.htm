California Roadkill Observation System API

 

Account Registration

 

An account creation form is available at the following URL: http://www.wildlifecrossing.net/california/user/register. After the registration request is submitted, an email containing an initial account password is mailed to the email address specified in the registration request.

Since handling this registration process, as it stands, completely within the app is not feasible, the recommended approach is to provide a "Registration" button that launches Safari with the above URL. An explanatory note could accompany the button, instructing the use to return to the app after the registration process is complete.

Since observations submitted without an authenticated session require filling out a captcha field, the app (for now) will require account creation and authentication to operate fully.

Login

 

Once an account has been created and the password obtained, an authenticated session may be created by issuing an HTTP request with the following form

POST /california/node HTTP/1.1

Host: www.wildlifecrossing.net

Content-Type: application/x-www-form-urlencoded

Content-Length: <length>

 

name=<username>&pass=<password>&op=Log+in&form_id=user_login_block

 

For example,

POST /california/node HTTP/1.1

Host: www.wildlifecrossing.net

Content-Type: application/x-www-form-urlencoded

Content-Length: 65

 

name=seantest1&pass=mypassword&op=Log+in&form_id=user_login_block

 

If successful, an HTTP 302 response containing cookies for the authenticated session will be returned. For example,

The HTTP client used must support cookies, as the session cookie values must be supplied in other API requests. Note that the response may set the same cookie twice. In such a case, the later cookie value must be used.

If the Login request does not succeed, an HTTP 200 request will be returned containing an error HTML page. There is no need for the app client to parse this page. It may be assumed (for now) that an HTTP 200 response indicates that the supplied username or password is invalid.

 

Get Form Token

 

In order to submit an observation, it is necessary to obtain a "form token. A form token may be requested by issuing an HTTP request of the following form.

GET /california/node/add/roadkill HTTP/1.1

Host: www.wildlifecrossing.net

Cookie: <cookie-list>

 

 

The <cookie-list> consists of a semicolon-separated list of the cookie name-value pairs obtain from the Login response. For example,

GET /california/node/add/roadkill HTTP/1.1

Host: www.wildlifecrossing.net

Cookie: SESS013c209a2abf719c50f37613053825dd=e4562b6b2d1f9095db334948d72850e5

 

 

If successful, an HTTP 200 response containing an HTML page will be returned. The form token is present in the page as the value of the "form_token" input field of Add Observation form. This value may be extracted using the following XSL statement.

<xsl:value-of select="/html/body//form[@action='/california/node/add/roadkill']//input[@name='form_token']/@value" disable-output-escaping="yes"/>

 

A complete XSL stylesheet for this operation would be as follows.

<?xml version="1.0" encoding="ISO-8859-1"?>

<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

<xsl:output method="text" encoding="utf-8" indent="no" omit-xml-declaration="yes" />

<xsl:template match="/">

 

<xsl:value-of select="/html/body//form[@action='/california/node/add/roadkill']//input[@name='form_token']/@value" disable-output-escaping="yes"/>

 

</xsl:template>

</xsl:stylesheet>

 

Leading/trailing whitespace must be trimmed from the resulting token value.

If the request is unsuccessful, the form_token input field will be absent from the returned page.

 

Field Taxonomy Reference Auto-complete

 

A list of field taxonomy references may be obtained by issuing an HTTP request with the following form

GET /california/nodereference/autocomplete/field_taxon_ref/<completion string> HTTP/1.1

Host: www.wildlifecrossing.net

 

For example,

GET /california/nodereference/autocomplete/field_taxon_ref/duck HTTP/1.1

Host: www.wildlifecrossing.net

The ID number assigned to the new observation is returned in the "Location" header of the response.

In the case that the request is not successful, an HTTP 200 response containing an error page is returned. Parsing that page to determine the exact error should not be necessary for a prototype app.
